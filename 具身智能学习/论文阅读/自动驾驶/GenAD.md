# GenAD: Generative End\-to\-End Autonomous Driving

2024\.02

！生成式  端到端

# 动机

![image](</具身智能学习/src/genad/image 4.png>)

生成式端到端自动驾驶框架与传统自动驾驶流水线的比较。大多数现有的方法都遵循一系列的**感知、预测和规划设计**。他们通常忽略了自车和其他智体之间的高层交互以及现实轨迹的结构先验。将自动驾驶建模为**未来生成的问题**，并在结构化潜轨迹空间中同时进行运动预测和自我规划。

> 通俗解释，就是之前的方法，在prediction部分预测agent怎么走，在planning的时候规划ego轨迹。
>
> 而agent或同时生成ego和agent的轨迹

# 方法

![image](</具身智能学习/src/genad/image 6.png>)

## Image to BEV

遵循BEVFormer来获得BEV特征。

使用卷积神经网络和特征金字塔网络从相机属兔中获取多尺度**图像特征，**然后进入可变形注意力块（由交错的自注意力层和可变形交叉注意力层组成），并将来自p个过去帧的BEV特征对准到当前坐标系中，并将它们连接为最终BEV特征B。

![image](</具身智能学习/src/genad/image 3.png>)

## BEV to map

遵循VAD，我们考虑**三类地图元素**（即车道分隔线、道路边界和行人过路处）。

![image](</具身智能学习/src/genad/image 7.png>)

## BEV to agent

获得agent tokens A 后，使用 3D 对象检测头 DA 解码每个 token a 的**位置、方向和类别**信息。

![image](</具身智能学习/src/genad/image 1.png>)

## Instance\-centric scene representation以实例为中心的场景表示

将自我标记e添加到学习到的agent token A中，构造一组实例标记 I = concat\(e, A\)

我们通过对实例令牌执行自我关注来实现自我车辆和其他代理之间的高阶交互：

![image](</具身智能学习/src/genad/image 9.png>)

为了执行准确的预测和规划，智能体和自我车辆都需要了解语义地图信息。因此，我们在更新的**实例token**和学习的**地图token**之间采用**交叉注意力**来获得地图感知的以实例为中心的场景表示：

![image](</具身智能学习/src/genad/image 5.png>)

学习到的**实例token**包含高阶代理\-自我交互，并了解学习到的语义图，这些语义图很紧凑，包含执行运动预测和轨迹规划所需的所有地图和实例信息。

## Trajectory

传统方法直接用解码器"硬猜"轨迹，但作者认为这太粗暴了。因为：

- 轨迹是高度结构化的（大部分是直线或平滑曲线，很少出现"之字形"）
- 存在不确定性（同一情境下可能有多种合理走法）

所以用 VAE 来学习一个轨迹的潜在分布，让模型学会"什么样的轨迹是合理的"。

![image](</具身智能学习/src/genad/image 8.png>)

我们使用**未来轨迹编码器**将**地面真实轨迹**映射到潜在轨迹空间，在那里我们使用高斯分布来建模轨迹的不确定性。然后我们使用门递归单元（GRU）来逐步预测潜在空间中的下一个未来，并使用解码器来获得显式轨迹。

![image](</具身智能学习/src/genad/image 2.png>)
